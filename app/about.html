BLE Mouse Controller

A CircuitPython project for continuous mouse movement and keyboard input, controllable wirelessly via Bluetooth from any browser with Web Bluetooth support.

Launch App GitHub

Hardware

Designed for the MDBT50Q-RX (nRF52840) running CircuitPython 10. The board acts as a USB HID device (mouse + keyboard) connected to a PC, phone, or tablet, while being controlled wirelessly from a second device over BLE UART.

Web App Features

  • Connect / disconnect to the board via BLE (requires Chrome or any Chromium-based browser)
  • Auto-movement: Start, Stop, Toggle — keeps the cursor active to prevent screensaver
  • Mouse controls: directional pad, custom X/Y move, left/middle/right click, scroll
  • Type Text — send text to be typed via USB HID on the connected PC
  • Cipher Text — AES-128-CTR encrypted text; the device decrypts and types it, protecting passwords over BLE
  • Keyboard Layout (Settings) — select English (US) or Italian to match the OS of the destination machine
  • Installable as a PWA on Android

BLE Commands

CommandAction
STARTEnable auto-movement
STOPDisable auto-movement
TOGGLEToggle auto-movement on/off
MOVE x yMove mouse by x,y pixels
CLICK [LEFT|RIGHT|MIDDLE]Mouse click
SCROLL nScroll n steps (positive = up)
TYPE textType text via USB HID keyboard
LAYOUT ENSwitch to English (US) keyboard layout
LAYOUT ITSwitch to Italian keyboard layout
CIPHER <b64>Decrypt AES-128-CTR payload and type it

Keyboard Layout

The layout setting on the dongle must match the OS keyboard layout of the machine it is plugged into — not the machine running the browser. If they differ, special characters (like @) will produce the wrong output.


The Italian layout requires keyboard_layout_win_it.mpy and keycode_win_it.mpy in the /lib folder on the CIRCUITPY drive (available from the Neradoc keyboard layouts releases).

Cipher — Secure Password Typing

The Cipher feature lets you type passwords over BLE without sending them in plaintext. The web app encrypts the text with AES-128-CTR using a pre-shared key, and the device decrypts and types it via USB HID.


The pre-shared password is stored in secret.txt on the device and in localStorage in the browser. Physical access to either defeats the protection.

Requirements

  • Chrome or any Chromium-based browser (Web Bluetooth required)
  • The app must be served over HTTPS — Web Bluetooth does not work over plain HTTP
  • A paired BLE device running the CircuitPython firmware
